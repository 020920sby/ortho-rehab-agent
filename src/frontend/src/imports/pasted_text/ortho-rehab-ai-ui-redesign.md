骨科康复AI助手 UI 重设计方案

一、整体设计核心原则与底层依据

本次设计围绕「AI驱动的骨科术后全周期个性化康复管理」核心定位，针对TKA等骨科术后康复患者的核心痛点（看不懂医嘱、康复动作不标准、恢复进度无感知、康复依从性差、异常风险难预警），确立5大核心设计原则，所有页面布局均围绕以下逻辑展开：

1. AI能力全链路前置：打破AI藏于后台的问题，让用户全程感知AI的核心价值，强化「AI专属康复管家」的产品心智

2. 核心操作路径最短：高频刚需功能一步直达，降低操作门槛，适配术后患者、中老年用户的操作能力

3. 医疗安全感优先：核心安全信息、风险预警置顶，用专业严谨的设计降低用户康复焦虑，同时满足医疗合规要求

4. 全链路正向反馈：用可视化、里程碑式设计，把长期枯燥的康复拆解为可感知的小目标，提升用户长期依从性

5. 全场景老年友好：大字体、大点击区域、低饱和视觉、简化输入逻辑，适配核心用户群体的使用习惯

基础视觉规范

• 主色调：专业医疗蓝 #2A79E6（传递信任、安全，低饱和不刺眼）

• 辅助色：安全绿 #48C774（正向反馈、正常状态）、预警橙 #FF9500（提醒、轻度异常）、危险红 #F53F3F（重度异常、风险提示）

• 字体：无衬线字体，正文≥14px，关键信息加粗放大，最小点击区域≥44px，避免误触

• 组件风格：圆角16px卡片式设计，用留白、轻阴影区分层级，减少冗余分割线，降低认知负担
二、分页面详细设计与布局依据

底部导航栏（全局固定）

保留5个tab，逻辑完全贴合用户核心使用闭环，图标+文字双展示，清晰无学习成本：
首页 → 打卡 → 训练 → 进度 → 我的
布局依据：用户完整使用路径为「打开APP看今日康复任务→记录身体状态→完成康复训练→查看恢复进度→管理个人/医嘱信息」，路径闭环，符合用户使用习惯。
页面1：首页（用户打开APP的第一触点，核心优先级最高）

从上到下布局设计

1. 顶部全局栏

◦ 左侧：产品名称「骨科康复助手」+ 骨骼logo

◦ 右侧：固定悬浮AI对话按钮（全页面常驻，点击一键发起AI咨询，如“膝盖肿了怎么办”“动作不标准怎么调整”）
布局依据：AI是产品核心，必须让用户在任何场景下都能一键触达，解决突发康复疑问、异常情况咨询的刚需，同时强化AI产品心智。
2. 用户核心信息&安全状态通栏卡片（视觉第一焦点）

◦ 左侧：问候语「早上好/下午好/晚上好，XXX」，下方核心标签「TKA右膝术后 · 第19天 · 亚急性期」

◦ 右侧：安全状态徽章（正常为绿底白勾「安全状态：一切正常」；异常为橙/红底色，直接展示核心异常项）

◦ 卡片底部小字：「AI已根据你的昨日打卡数据，更新今日个性化康复方案」
布局依据：术后患者最核心的需求是「安全感」，打开APP第一眼就明确自身康复阶段、当前是否安全，快速建立信任；同时直接告知用户AI的服务价值，让用户感知到AI在持续为其服务，而非冰冷的工具。
3. 今日核心康复任务卡片（第二视觉焦点，核心操作一步直达）

◦ 标题「今日康复任务」，右侧「去打卡」快捷按钮

◦ 全屏进度条：可视化展示「已完成2/5项」，下方标注预计完成总时长

◦ 双核心按钮并排（占满卡片宽度）：

◦ 左侧主按钮（主色填充）：「开始今日训练」，点击直接跳转训练页，加载今日AI定制方案

◦ 右侧次按钮（描边）：「记录今日状态」，点击直接跳转打卡页

◦ 底部折叠展示2个核心训练项，附带「开始」快捷按钮
布局依据：用户打开APP的核心动作就是「完成训练」和「打卡记录」，将两个核心操作做成大按钮，一步直达，减少跳转步骤，极大提升康复依从性；进度条可视化给用户明确的目标感，降低康复的心理负担。
4. 今日身体核心指标卡片

◦ 标题「今日身体状态」，分3行展示核心指标，每行附带可视化进度条：

◦ 疼痛指数：X/10，标注「轻度/中度/重度」，进度条采用绿-黄-红渐变

◦ 膝关节屈曲：X°，标注「目标XX°」，进度条展示距离目标的完成进度

◦ 行走能力：当前状态（单拐辅助/自主行走等），进度条匹配康复阶段进度
布局依据：用可视化方式降低医疗数据的理解门槛，用户一眼就能看懂自身恢复情况，同时将指标与目标绑定，给用户明确的康复方向。
5. 高频提醒2列网格卡片

◦ 2个卡片并排，不占用过多空间，清晰明了：

◦ 左侧：「下次复诊」，展示日期、科室、医生，附带「设置提醒」按钮

◦ 右侧：「用药提醒」，展示今日待服药物，附带「去查看」按钮
布局依据：复诊、用药是术后患者高频刚需，且极易遗漏，放在首页提升触达率，避免用户错过关键医疗事项；2列布局适配中老年用户视觉习惯，无阅读压力。
6. AI核心能力快捷入口卡片

◦ 标题「AI康复助手」，下方3个横向滚动快捷按钮：

◦ 「上传出院小结」：点击直接上传，AI解析生成专属康复方案

◦ 「AI康复咨询」：点击进入对话页面，一对一咨询康复问题

◦ 「异常情况预警」：点击查看AI分析的异常项与应对建议
布局依据：把产品核心AI能力做成快捷入口，让用户明确知道AI能提供的服务，解决“不知道AI能干嘛”的问题，同时降低AI功能的使用门槛。
7. 底部固定合规声明

◦ 小字固定展示：「本应用为AI辅助工具，不能替代专业医疗判断，如遇紧急情况，请立即拨打120就医」
布局依据：医疗类产品强制合规要求，全程可见，规避合规风险。
页面2：打卡页面（AI方案调整的核心数据来源，降低填写门槛为核心）

从上到下布局设计

1. 顶部标题栏

◦ 主标题「每日康复打卡」，副标题「记录今日恢复情况，AI将为你自动调整康复计划」
布局依据：明确告知用户打卡的核心价值，让用户知道“打卡不是为了记录，而是为了让AI优化更适合自己的方案”，提升用户打卡意愿。
2. 疼痛程度评估模块（第一优先级，用户最直观的感受）

◦ 标题「今天的疼痛程度」，采用「表情+滑块」的设计，0无痛（笑脸）到10剧痛（哭脸），滑块实时显示数字，下方同步标注「无痛/轻度/中度/重度/剧痛」
布局依据：表情化设计极大降低理解门槛，哪怕是不识字的老年用户，也能通过表情准确选择自身疼痛程度；滑块操作简单，无需打字，大幅降低填写成本，提升打卡完成率。
3. 核心康复指标填写模块

◦ 2个输入项，均默认填充昨日数据，用户仅需修改变化内容：

◦ 「关节活动度」：输入框，提示语「膝关节屈曲XX度，伸展XX度」

◦ 「行走能力」：下拉选择框，选项为「卧床/助行器/双拐辅助/单拐辅助/自主行走」，默认选中昨日选项
布局依据：默认值填充减少用户输入成本，下拉选择替代手动输入，既提升填写效率，也能保证数据标准化，方便AI精准分析。
4. 异常症状与康复感受模块

◦ 优先展示「不适症状（选填）」：标签式多选按钮，选项为「小腿肿胀、切口发红、发热、小腿疼痛、关节弹响、无法承重、其他」，点击选中后高亮变色

◦ 下方展示「今天的康复感受（选填）」：多行输入框，提示语「可以写下今天的康复感受、遇到的问题，AI会为你解答」
布局依据：不适症状是AI判断医疗风险的核心数据，优先级高于主观感受，引导用户优先填写，避免遗漏风险项；标签式多选无需打字，降低填写门槛，非强制输入也能减少用户的打卡心理负担。
5. 底部固定提交按钮

◦ 全屏宽度主色按钮：「提交并生成今日康复计划」，按钮下方小字「AI将根据你提交的信息，个性化调整今日训练的强度与内容」
布局依据：固定在页面底部，用户填写完成后无需滑动即可点击；再次明确提交后的AI价值，提升用户提交意愿，完成「数据输入→AI输出方案」的闭环。
页面3：训练页面（AI康复方案的落地页，核心是降低动作执行门槛）

从上到下布局设计

1. 顶部标题栏

◦ 主标题「今日康复训练」，副标题「AI个性化定制 · 完成所有训练，坚持就是胜利！」，右侧附带「训练指南」按钮（点击查看AI生成的今日训练注意事项）
布局依据：再次强化AI个性化的核心价值，让用户知道方案是专属定制而非通用模板，提升用户的重视程度与执行依从性。
2. 全局训练进度条

◦ 全屏宽度进度条，标注「已完成1/5项」，下方小字标注「预计总时长XX分钟」
布局依据：给用户明确的目标感与时间预期，减少康复的心理负担，进度实时更新，给用户持续的正向反馈。
3. 训练项列表（核心内容，按康复顺序排列）

◦ 每个训练项为独立通栏卡片，分「已完成/未完成」两种状态：

◦ 已完成项：左侧绿色对勾+序号，标题加粗，下方展示时长、组数次数、动作要点，整体灰色置灰，标注「已完成」

◦ 未完成项：左侧空白圆圈+序号，标题加粗，第一行展示时长+组数次数，第二行展示动作要点（图标+短句），第三行用黄色警示图标标注禁忌/注意事项，右侧附带「开始」主色按钮（点击进入动作详情页）
布局依据：按序号排序引导用户按流程完成，避免混乱；动作要点、注意事项直接展示在列表中，用户无需跳转即可看到核心要求，降低操作门槛；每个动作独立入口，支持中途退出、随时继续，灵活性更高，提升训练完成率。
4. 底部固定提交按钮

◦ 未全部完成时：灰色不可点击按钮，标注「还剩X项训练待完成」

◦ 全部完成后：主色可点击按钮，标注「提交训练完成记录」，点击后提交数据，AI同步更新康复进度，跳转回首页并给出完成正向反馈
布局依据：引导用户完成全部训练，避免遗漏；全部完成后给出明确的提交入口，形成「方案→执行→数据同步」的完整闭环。
补充：单动作详情页设计

每个动作详情页必须包含：循环播放的动作演示视频/GIF、AI语音实时指导、自动计时/计数功能、休息提醒、紧急暂停按钮、一键AI咨询入口
设计依据：解决用户“不会做、做不对”的核心痛点，视频+语音指导降低动作难度，避免错误动作导致二次损伤，AI全程陪伴提升用户安全感。
页面4：进度页面（康复效果可视化，核心是给用户持续正向反馈）

从上到下布局设计

1. 顶部标题栏

◦ 主标题「我的康复进度」，副标题「AI全程跟踪 · 每一步进步都值得被记录」，右侧附带「生成康复报告」按钮（点击AI生成阶段性康复报告，可导出分享给主治医生）
布局依据：强化AI的跟踪分析能力，同时解决用户“复诊时不知道怎么和医生描述恢复情况”的核心痛点，提升产品的医疗实用价值。
2. 核心康复指标趋势图模块（视觉第一焦点）

◦ 支持左右滑动切换2个tab：「疼痛评分趋势」「关节活动度趋势」

◦ 疼痛评分：柱状图，支持日/周维度切换，下方绿色字体标注正向变化，如「疼痛从7分降至4分，下降43%」

◦ 关节活动度：折线图，按术后周数展示屈曲角度，同步标注目标值参考线，清晰展示距离目标的进度
布局依据：图表可视化比纯数字更直观，用户一眼就能看到自身恢复趋势，哪怕是微小的进步也能被感知，给用户持续的正向反馈，解决康复周期长、易放弃的痛点。
3. 康复里程碑模块

◦ 标题「康复里程碑」，横向滚动卡片，分「已完成」「待完成」两种状态：

◦ 已完成：绿底卡片+对勾，如「术后第1周：完成首次直腿抬高」「术后第2周：膝关节屈曲达到90°」

◦ 待完成：灰底卡片+锁图标，如「术后第4周：膝关节屈曲达到110°」「术后第6周：脱拐自主行走」
布局依据：把漫长的康复周期拆解为一个个可实现的小里程碑，用户每完成一个就能获得强烈的成就感，大幅提升长期康复的依从性。
4. 每日康复记录时间线模块

◦ 标题「每日康复记录」，按时间倒序排列，时间线样式展示：

◦ 每条记录包含：日期、疼痛评分、屈曲角度、训练完成情况，附带「查看详情」按钮，可查看当日完整打卡与训练数据

◦ 完成全部训练的日期，标注绿色对勾高亮
布局依据：时间线形式让用户清晰看到自己的康复历程，感受到坚持的意义；同时方便回溯历史数据，为AI调整方案提供依据，也可作为复诊时的参考资料。
页面5：我的页面（个人信息与核心设置，优先级按功能重要性排序）

从上到下布局设计

1. 顶部用户信息通栏卡片

◦ 左侧：用户头像、姓名，下方核心康复标签「TKA右膝术后 · 亚急性期」

◦ 右侧：「编辑个人信息」按钮
布局依据：打开页面第一眼明确个人专属属性，同时方便用户快速修改个人基础信息。
2. AI核心能力配置模块（最高优先级）

◦ 标题「AI康复核心设置」，列表式布局，每项包含图标、标题、副标题、跳转箭头：

◦ 「上传医嘱/出院小结」：副标题「AI自动解析，生成个性化康复方案」

◦ 「AI康复方案设置」：副标题「调整训练强度、提醒时间等个性化配置」

◦ 「异常预警设置」：副标题「设置异常情况的提醒阈值与通知方式」
布局依据：把产品核心的AI能力配置放在最顶部，强化AI核心定位，让用户快速找到方案调整入口，让康复方案更适配自身情况。
3. 康复管理工具模块（高频刚需功能）

◦ 标题「康复管理工具」，列表式布局：

◦ 「用药提醒」：副标题展示当前待服药物

◦ 「复诊提醒」：副标题展示最近复诊信息

◦ 「康复知识库」：副标题「TKA术后指南 · 饮食建议 · 常见问题」
布局依据：把高频使用的康复管理工具分类聚合，清晰明了，用户无需在首页寻找，即可快速完成设置。
4. 系统设置模块（最低优先级）

◦ 标题「系统设置」，列表式布局：

◦ 「通知管理」：设置打卡、训练、用药、复诊的提醒方式

◦ 「隐私设置」：管理个人数据与隐私权限

◦ 「关于我们」：产品介绍、合规信息、联系方式
布局依据：系统设置为低频功能，放在页面最底部，不干扰用户的核心操作。
5. 底部固定合规声明：与首页保持一致，满足医疗合规要求。
三、核心设计亮点与差异化价值

1. AI能力从“后台”走到“前台”：全链路贯穿AI入口与价值告知，彻底解决了初版设计中“AI感知弱、用户不知道AI能干嘛”的问题，真正把AI做成产品的核心，而非附加功能。

2. 完全贴合康复患者的用户旅程：从打开APP的安全感建立，到打卡、训练的低门槛操作，再到进度页的正向反馈，全链路匹配用户康复过程中的心理与行为需求，从根源上提升用户康复依从性。

3. 医疗专业性与易用性完美平衡：既满足了医疗产品的严谨性、合规性要求，又通过老年友好设计、简化操作逻辑，让不同年龄、不同文化程度的用户都能轻松上手。

4. 全链路风险防控：从打卡页的异常症状收集，到首页的安全状态展示，再到全局的AI紧急咨询入口，全流程覆盖术后康复的风险预警，最大程度保障用户康复安全。
